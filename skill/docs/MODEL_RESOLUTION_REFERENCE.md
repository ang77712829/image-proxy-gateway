# 各渠道模型分辨率与尺寸参考

> 本文档用于给 Agent 做尺寸选择。结论分成三类：
> - **官方明确支持**：有公开文档依据；
> - **网关当前实现**：代码里实际允许或实际透传的范围；
> - **项目实测**：我们实测可用，但不代表平台永久承诺。

## 1. Kolors（硅基流动）

**官方明确支持固定尺寸集合**，不支持任意分辨率。网关当前只把这些尺寸传给 `Kwai-Kolors/Kolors`；如果传入其他尺寸，会回退到 `1024x1024`。

| 尺寸 | 比例 | 方向 | 备注 |
|---|---:|---|---|
| `1024x1024` | 1:1 | 方形 | 默认 |
| `960x1280` | 3:4 | 竖图 | 适合封面、海报 |
| `768x1024` | 3:4 | 竖图 | 更轻量 |
| `720x1440` | 1:2 | 长竖图 | 长图 / 竖版视觉 |
| `720x1280` | 9:16 | 竖屏 | 手机竖屏 |

说明：硅基流动官方图片接口文档也列出了 Qwen-Image 的一组推荐尺寸，但当前网关的 `qwen` 别名走的是 ModelScope，不是 SiliconFlow Qwen，所以不要把 SiliconFlow Qwen 的尺寸误用到当前 `qwen` 别名上。

## 2. ModelScope（魔搭）

当前魔搭通道模型：

| 别名 | 模型 | 当前网关尺寸处理 | 建议 |
|---|---|---|---|
| `qwen` | `Qwen/Qwen-Image-2512` | 当前 Provider 暂不透传 `size` | 默认按 `1024x1024` 使用 |
| `flux` | `black-forest-labs/FLUX.1-Krea-dev` | 当前 Provider 暂不透传 `size` | 默认按 `1024x1024` 使用 |
| `z-image` | `Tongyi-MAI/Z-Image` | 当前 Provider 暂不透传 `size` | 默认按 `1024x1024` 使用 |
| `z-turbo` | `Tongyi-MAI/Z-Image-Turbo` | 当前 Provider 暂不透传 `size` | 默认按 `1024x1024` 使用 |

说明：这里不能写成“魔搭最高支持多少”，因为 ModelScope API-Inference 不同模型的参数兼容情况不完全一致，而且当前网关没有把 `size` 传给 ModelScope。需要精确尺寸控制时，后续应为具体模型补参数映射并实测。

## 3. Pollinations

Pollinations 旧公共图片接口通过 URL 参数传 `width` / `height`，新版 OpenAI-compatible 接口使用 `size`。

| 项目 | 说明 |
|---|---|
| 参数 | `width` / `height` 或 `size` |
| 默认 | `1024x1024` |
| 建议范围 | `512x512` 到 `2048x2048` 之间 |
| 过大尺寸 | 可能超时、失败或质量下降 |

说明：Pollinations 文档示例里出现过 `1920x1080` 这类自定义尺寸，但没有稳定公开承诺“任意大尺寸都可靠”。Agent 默认不要超过 2048 边长。

## 4. Agnes Image（`agnes-image-2.1-flash` / `agnes-image-2.0-flash`）

Agnes 图片模型支持自定义 `size` 字符串。官方公开文档能确认 Agnes Image 2.1 支持文生图和图生图；具体最大尺寸我没有找到足够硬的官方公开上限，所以这里按**项目实测 + 网关建议**写。

| 项目 | 说明 |
|---|---|
| 参数 | `size`，格式 `WIDTHxHEIGHT` |
| 推荐默认 | `1024x1024` 或 `1024x768` |
| 项目实测可用 | `2048x1536` |
| 项目实测可用 | `2048x1024` |
| `2048x2048` | 实测容易超时，不建议默认使用 |

建议：

- 普通图：`1024x1024`
- 横向封面 / 产品图：`1024x768` 或 `1536x1024`
- 高清横图：`2048x1536`
- 不要把 `2048x2048` 作为默认高清方案

## 5. Agnes Video（`agnes-video-v2.0`）

当前网关代码和项目实测按下面范围处理：

| 参数 | 当前网关范围 | 默认值 | 说明 |
|---|---:|---:|---|
| `width` | `256–2048` | `1152` | 宽度 |
| `height` | `256–1536` | `768` | 高度 |
| `num_frames` | `9–441`，且必须满足 `8n+1` | `121` | 帧数 |
| `frame_rate` | `1–60` | `24` | 帧率 |

常用时长换算：

| num_frames | frame_rate | 约等于 |
|---:|---:|---:|
| 81 | 24 | 3.4 秒 |
| 121 | 24 | 5.0 秒 |
| 161 | 24 | 6.7 秒 |
| 241 | 24 | 10.0 秒 |
| 441 | 24 | 18.4 秒 |

结论：你说的“只支持 441 帧，也就是 18s”基本对；更准确地说，当前网关上限是 **441 帧**，在默认 `24fps` 下约 **18.4 秒**。如果换成 `25fps`，则约 `17.6 秒`。

## 6. OpenAI-compatible Image（`gpt-image-2` / `openai-image`）

这是可选显式渠道，不进入默认免费链。

| 模型/渠道 | 尺寸说明 |
|---|---|
| GPT 图片模型标准尺寸 | `1024x1024`、`1536x1024`、`1024x1536` |
| `gpt-image-2` | 官方文档写明支持任意 `WIDTHxHEIGHT` 字符串；宽高需要是 16 的倍数，比例在 1:3 到 3:1 之间；超过 `2560x1440` 属于实验性，最大 `3840x2160` |
| 兼容第三方 OpenAI Image | 以对应服务商实际支持为准 |

建议：如果接的是真正 OpenAI `gpt-image-2`，可以按官方约束使用更灵活尺寸；如果接的是第三方兼容服务，先用 `1024x1024`、`1536x1024`、`1024x1536` 这三种更稳。

## Agent 快速选择建议

| 场景 | 推荐渠道 | 推荐尺寸 |
|---|---|---|
| 日常通用图 | `kolors` | `1024x1024` |
| 竖屏海报 | `kolors` | `960x1280` 或 `768x1024` |
| 手机竖屏图 | `kolors` | `720x1280` |
| 中文海报 / 带字图 | `qwen` | ModelScope 侧决定尺寸，网关当前不强传 |
| 写实人像 | `z-turbo` | ModelScope 侧决定尺寸，网关当前不强传 |
| 高清横向图 | `agnes-2.1` | `2048x1536` |
| Agnes 普通图 | `agnes-2.1` | `1024x1024` 或 `1024x768` |
| 视频默认 | `agnes-video-v2.0` | `1152x768`，`121` 帧 |
| 视频高清 | `agnes-video-v2.0` | `2048x1536`，建议先用 `81` 或 `121` 帧测试 |
| 视频最长 | `agnes-video-v2.0` | `441` 帧，24fps 下约 18.4 秒 |
| OpenAI 付费高质量 | `gpt-image-2` | 先用 `1024x1024` / `1536x1024` / `1024x1536` |
